1. Overriding Inline Styles

Inline styles (written directly in the HTML style attribute) usually have the highest priority. However, !important in the CSS file can force an override.

I should be RED (Inline), but !important makes me GREEN.

2. Overriding Cascade Order

Normally, if two rules have the same specificity, the one defined last wins. !important changes this, allowing an earlier rule to win.

I am BLUE because of !important, even though RED is defined later.

3. Forcing a Utility Class

Sometimes you need to guarantee an element is hidden or styled a certain way, regardless of other conflicting styles. This box has display: flex inline, but the class hides it.

You cannot see me because of display: none !important.
