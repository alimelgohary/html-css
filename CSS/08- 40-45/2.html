    .one {
        box-sizing: content-box;
        width: 300px;
        height: 50px;
        padding: 10px;
        border: 5px solid red;
    }
    .two {
        box-sizing: border-box;
        width: 300px;
        height: 50px;
        padding: 10px;
        border: 5px solid blue;
    }
    * {
        box-sizing: border-box;
    }

1. The Problem (content-box)

By default, if you set a width of 300px, adding padding (10px) and borders (5px) increases the total width to 330px. This often breaks layouts where elements are meant to sit side-by-side.

Total Width: 330px

2. The Solution (border-box)

Using box-sizing: border-box; forces the element to stay at the specified 300px width. The padding and border push the content inward rather than expanding the box outward.

Total Width: 300px

3. Global Reset (Best Practice)

It is standard practice to apply box-sizing: border-box; to all elements using the universal selector (*) to make sizing intuitive across the entire project.

(The global rule is applied in the CSS file)
