        -ms-border-radius: 10px;     /* Internet Explorer */
        -o-border-radius: 10px;      /* Old Opera */
    }

1. The Role and Format of Vendor Prefixes

Vendor Prefixes are special prefixes that browsers (vendors) attach to CSS properties to enable support for new or experimental features before they are fully standardized. This allows developers to use new features while they are still being tested.

The instructor demonstrates the old, full syntax for a property like border-radius:

  • -webkit-: Chrome, Safari, and newer versions of Opera.
  • -moz-: Mozilla Firefox.
  • -ms-: Microsoft Internet Explorer and Edge (older versions).
  • -o-: Older versions of Opera.
Prefixed Box

2. Modern Usage and The Standard Property

It is crucial to always list the standard, unprefixed property last. This ensures that when the browser fully supports the feature without a prefix, it will override any prefixed version above it, making your code future-proof.

The video notes that for most modern properties, developers rely on build tools (like task runners) or VS Code extensions to add prefixes automatically, as manually managing them for all modern browsers is no longer necessary.
