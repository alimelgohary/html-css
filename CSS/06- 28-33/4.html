1. Default Stacking Order (Based on HTML Source)

Without z-index, positioned elements stack based on their order in the HTML. The last element (Blue Box 3) is on top.

2. Controlling Stacking with Positive z-index

A higher z-index value brings the element forward. Purple (10) is on top, followed by Orange (5), then Cyan (1).

3. Pushing Elements Back with Negative z-index

A negative value (e.g., -1) pushes the element behind its parent's content, provided the parent has its own stacking context.

Parent Content (Z-Index 0) is ON TOP of the pink box.

4. z-index Only Works with Positioned Elements

The yellow box has z-index: 1000; but is ignored because its position is static (default). The blue box with z-index: 1; is correctly positioned on top of the yellow box.
